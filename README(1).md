# 🪞 Semantic Mirrors

**Exploring how humans and LLMs understand "oppositeness" in embedding space.**

## Project Overview

This project investigates what it means for a sentence to be the "opposite" of another — and whether humans and LLMs agree. 

The original sentence is:
> *The quick brown fox jumps over the lazy dog*

This sentence was chosen because it uses every letter of the English alphabet, is simple enough for all survey respondents to understand, and is long enough to generate meaningful word-level analysis.

**26 human participants** were surveyed and asked to provide an opposite word for each word in the sentence. Their responses were assembled into 26 "opposite sentences." A **ChatGPT-generated** opposite was also collected. All sentences were vectorized using OpenAI's `text-embedding-3-small` model and visualized using t-SNE and UMAP.

## Key Findings

- The sentence **farthest** from the original in embedding space was generated by a **human**, not the LLM.
- The **LLM** was more similar to the original than ~81% of human responses — suggesting it approximates a "common ground" human understanding rather than maximizing semantic distance.
- Human responses show high **variance**, reflecting that "oppositeness" is subjective and context-dependent.
- Even words with near-consensus opposites (dog→cat, quick→slow) show scatter when assembled into full sentences.
- LLMs appear to **align with perceived human expectations** rather than their own embedding-space logic — a constraint imposed by the word-level format of the prompt.

## Running the Notebook

1. Open `Semantic_Mirrors.ipynb` in Google Colab
2. Add your OpenAI API key to Colab Secrets as `Secret-Key`
3. Run all cells in order
4. Upload `human_survey_responses.csv` when prompted

## Running the Streamlit App

```bash
pip install -r requirements.txt
export OPENAI_API_KEY=your_key_here
streamlit run app.py
```

> **Note:** The app requires `data/sentence_embeddings.pkl` to be present. Run the notebook first to generate it.

## Limitations

- The word-level prompt format constrains LLM responses — an LLM operating freely in embedding space might produce far more "opposite" outputs.
- Asking humans for "opposite words" introduces cultural and linguistic bias.
- With only 26 respondents, human variance estimates are approximate.

## Tech Stack

- **Embeddings:** OpenAI `text-embedding-3-small` (1536 dimensions)
- **Dimensionality reduction:** t-SNE, UMAP
- **Visualization:** matplotlib, plotly
- **App:** Streamlit
